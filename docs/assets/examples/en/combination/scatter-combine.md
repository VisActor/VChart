# Custom Single Row Multi-Column Layout for Combination Charts

Combination charts often require a custom layout for arranging different series.

## Key Configuration

- `type: 'common'` declares the chart type as a combination chart
- `layout` attribute declares the custom layout for the combination chart

  - `layout.type` attribute declares the layout type, `grid` for the row-column layout
  - `layout.col` attribute declares the number of columns (Note: All independent elements in the chart need to occupy a column separately, such as data axes or other components, and chart series need to occupy a separate column)
  - `layout.row` attribute declares the number of rows (Note: Same as above)
  - `layout.col` attribute specifies the column width, supporting customization with `{ index: xx, size: xx }`, where `index` indicates the index of the column, and `size` indicates the column width
  - `layout.row` attribute specifies the row height, supporting customization with `{ index: xx, size: xx }`, where `index` indicates the index of the row, and `size` indicates the row height
  - `layout.elements` attribute declares the layout unit ID for binding data series and layout units. Declare it as `{modelId: xx, row: xx, col: xx}`, where `modelId` represents the ID name of the layout unit, and `row` and `col` represent the index of the row and column where the layout unit is located, respectively.

- `region` attribute declares the data region for subsequent binding, and the layout units using the same data need to bind the same `regionId`, such as the data axis and the corresponding chart series in the layout units. Declare it as `id: xx`, where `id` represents the data region ID.

- `series` attribute declares different chart series, and in the series configuration, `regionIndex` is used to bind the data region where the chart series is located. `id` is used to bind the axis in the layout unit, corresponding one-to-one with the `modelId` in `layout.elements`. (Since there is a correspondence between the data region and the `modelId` in `layout.elements`, there is no need to declare `id` here)

- `axes` attribute declares different coordinate axis components, and the `regionIndex` and `id` attributes are the same as above.
